# Zarinpal Production Setup

Logistic Plus uses direct Zarinpal REST calls from `server.js`; the official SDK is not required for the current integration.

## Required Production Settings

Set these on the Liara app `logisticplus`:

```text
APP_PUBLIC_URL=https://logisticplus.ir
ZARINPAL_SANDBOX=false
ZARINPAL_MERCHANT_ID=<live merchant id from Zarinpal dashboard>
ZARINPAL_TIMEOUT_MS=10000
```

Keep the normal production settings in place too:

```text
NODE_ENV=production
RATE_LIMIT_STORE=postgres
TRUST_PROXY=true
DOCUMENT_STORAGE_DIR=storage/documents
```

The callback URL generated by the app is:

```text
https://logisticplus.ir/api/billing/zarinpal/callback
```

## Flow

1. `/signup` creates a signup request, pending organization/subscription, invoice, and `billing_payments` row.
2. `POST /api/billing/payments/:id/start` sends the live request to Zarinpal when `ZARINPAL_SANDBOX=false`.
3. The API stores the returned `authority` and redirects the user to `https://www.zarinpal.com/pg/StartPay/<authority>`.
4. Zarinpal returns to `/api/billing/zarinpal/callback`.
5. The callback verifies the payment using the original DB amount, then updates payment, invoice, receipt, signup, organization, and subscription state in an idempotent transaction.

## Verification

Use a controlled low-value signup to confirm:

- Payment start returns a gateway URL beginning with `https://www.zarinpal.com/pg/StartPay/`.
- A cancelled/failed payment redirects to `/signup/pending?payment=failed`.
- A paid transaction redirects to `/signup/pending?payment=paid`.
- Duplicate callbacks do not create duplicate receipts or invoices.

Do not manually call the production callback with `Status=OK` unless the `Authority` came from a real Zarinpal payment attempt.
